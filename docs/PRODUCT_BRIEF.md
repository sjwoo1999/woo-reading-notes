# Product Brief (1-pager)

## 문제

독서 기록은 쌓이지만 **개념 간 연결**과 **회고·재사용**이 어려움.
- 개별 노트는 고립됨 (링크/관계 부재)
- 검색/탐색이 선형적 (시간 순)
- 재독·재사용 힌트 부재

## 가치 제안

독서 노트를 **노트+링크+그래프**로 구조화하여:
- 개념 간 관계 가시화
- 탐색/발견 효율 ↑
- 학습 피드백 루프 강화
- 창작물(에세이·글쓰기) 아이디어 도출 가속화

## 대상 사용자

- **1차**: 학습자, 연구자, 에세이 작성자 (노트 → 아이디어 구체화)
- **2차**: 큐레이터, 북블로거 (공유/출판)

## 핵심 지표 (북극성)

**주간 그래프 탐색 세션률 ≥ 30%**
- 정의: (1주일 내 그래프 뷰 진입한 활성 사용자 / 전체 활성 사용자)

## 초기 성공 KPI (Release-1, 2주 후)

| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| D7 리텐션 | ≥ 25% | 활동 > 0 인 사용자의 7일 리텐션 |
| 노트당 링크 | ≥ 1.2 | Avg(links / notes) |
| 그래프 세션 | ≥ 30% | 주간 그래프 뷰 진입률 |
| 평균 체류 시간 | ≥ 45초 | 그래프 뷰 평균 세션 시간 |
| 오류율 | < 1% | 5xx 에러 / 전체 요청 |
| TTI (데스크톱) | < 2.5s | Time to Interactive, Lighthouse |

## 범위 (Release-1)

### 포함 (4주)
- ✅ **노트 관리**: 책, 개념, 인용 타입
- ✅ **링크/관계**: 양방향 링크, 관계 타입 (relates_to, supports, contradicts, inspired_by)
- ✅ **그래프 뷰**: 인터랙티브, 필터 (타입/태그), 검색 연동
- ✅ **검색**: 제목·본문·태그 검색 (Postgres FTS)
- ✅ **리마인더**: 1-3-7-30일 간격 반복 복습

### 제외 (추후)
- ❌ 추천 (R-2)
- ❌ 소셜/공유 (R-2)
- ❌ 플러그인/API (R-3)
- ❌ 협업 편집 (R-4)

## 주요 가정 (리스크)

| 가정 | 영향 | 완화책 |
|------|------|--------|
| 사용자가 그래프를 탐색할까? | 높음 | 초기 온보딩에서 그래프 가치 설명, 리마인더로 유도 |
| 대규모 그래프(1k+노드)도 성능을 유지할까? | 높음 | 조기 성능 테스트(주 4), 캐싱/LOD 전략 |
| 한글 검색·링크 파싱이 정확할까? | 중간 | Mecab/KoNLPy 테스트, 사용자 피드백 반영 루프 |
| 기존 데이터 마이그레이션이 완벽할까? | 중간 | 스크립트 검증, 샘플 10개 노트 수동 검증 |

- **Beta** (1주): 피드백 수집, 성능 최적화
- **Stable** (2주): 문서화, 공개 출시 (GitHub)
